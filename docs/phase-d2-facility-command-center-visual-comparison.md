# Phase D.2 Facility Command Center Visual Comparison

Before:
- D.1 rendered the Facility Workspace as a conventional dashboard.
- Widgets appeared as mostly similar bounded cards.
- Most drill-downs were links to separate pages.
- The executive state had similar weight to individual metric widgets.
- Priority items were part of the grid rather than the main work surface.

After:
- The screen is organized as a command center:
  - Command Header.
  - Situation Overview.
  - Operational Pulse.
  - Intervention Queue.
  - Context Panel.
  - Action Center.
- The executive state is the strongest visual element.
- Four metrics are merged into a compact pulse rail.
- Priority items are compact operational rows.
- Notes and corrective actions can be inspected inside the same workspace.
- Escalation, form, and activity previews stay in context.
- Full pages are secondary links.

Screenshot artifacts:
- `docs/screenshots/phase-d2/` is reserved for before/after captures.
- Automated capture depends on a runnable local API/session; if unavailable locally, CI/PR review should attach captures from the review environment.
