# ADR-002: Microsoft Entra ID مع تخزين النطاقات في SQL

## الحالة

مقبول — المرحلة A

## السياق

المطلوب مصادقة مؤسسية قابلة للربط مع Entra ID، مع صلاحيات ونطاقات تنظيمية دقيقة لا تُدار فقط من مزود الهوية.

## القرار

- المصادقة عبر JWT من Microsoft Entra ID (`Microsoft.Identity.Web`).
- الأدوار والصلاحيات و`UserScope` تُخزَّن في SQL Server وتُحمَّل بعد التحقق من التوكن.
- اختبارات التكامل تستخدم `TestAuthHandler` دون استدعاء Entra.

## العواقب

- تسجيل الدخول التفاعلي يتطلب إعداد Tenant/Client.
- فصل واضح بين هوية الدخول وتفويض التشغيل المحلي.
