# Phase D.2 Facility Command Center Visual System

This phase introduces command-center CSS variables above the existing Baseera design tokens:
- `--command-bg`
- `--command-surface`
- `--command-surface-raised`
- `--command-surface-muted`
- `--command-border-subtle`
- `--command-border-strong`
- `--command-text-primary`
- `--command-text-secondary`
- `--command-accent`
- `--command-accent-soft`
- `--command-critical`
- `--command-warning`
- `--command-attention`
- `--command-stable`
- `--command-info`
- `--command-shadow`
- `--command-focus`

Design intent:
- Calm institutional operations surface.
- Fewer identical white cards.
- Status and priority use color; secondary data is neutral.
- Executive state is visually dominant.
- Priority queue is a compact operational list, not repeated cards.
- Context panel is a raised workspace surface for investigation and action.

Visual changes from D.1:
- Header is a command surface, not a widget card.
- Four KPI cards became an operational pulse rail.
- Priority items became compact rows with priority bands.
- Detail pages became in-context previews where supported.
